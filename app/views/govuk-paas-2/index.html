Deploy and run government applications

GOV.UK Platform as a Service (PaaS) is a quick, easy and cheap way to deploy and host government applications. The PaaS manages the platform infrastructure, so you can focus on building your digital services.

Ask for a trial account

You’ll be able to deploy test applications in a safe environment and test the features of the command line tool.

Benefits of GOV.UK PaaS

Using GOV.UK PaaS means teams can deploy and run applications without having to build and manage their own infrastructure. This avoids unnecessary repetition across government, with infrastructure and webops engineers spending less time solving the same problems.

Service teams can:

  • save time and money which would have been spent on setting up a hosting infrastructure
  • deploy and run applications written in a range of languages and frameworks
  • rapidly scale applications
  • easily manage their applications
  • get 24 hour platform-level support
  • get started easily using a simple procurement process

What it is

GOV.UK PaaS is hosted by Amazon Web Services (AWS), run from AWS’s European data centre in Ireland. The PaaS is:

  • resilient since it operates across three independent availability zones
  • able to support a PostgreSQL database service (powered by AWS)
  • accredited for handling information classified as Official
  • built on an open source platform (Cloud Foundry) and deployed to a public cloud
  • flexible since applications aren’t locked into one cloud provider

The PaaS is already being used by the GOV.UK Trade Tariff live service, as well as several services building prototypes.

Find out more about GOV.UK PaaS. This includes how it’s built, run and supported, as well as how to use it.

What you pay for

You pay for what you use, like hosting, compute resource and backing services such as a database. The Government Digital Service will pay for building and maintaining the platform.

The overall cost depends on factors such as the number of environments you deploy to, the size and number of your applications and whether they need to have high availability.

Criteria for deploying an application to the PaaS

Your application can run on the PaaS if you:

  • follow the twelve-factor application principles - this will be the case if your app was written to be deployed to another PaaS, such as Heroku
  • don’t require any backing service apart from a PostgreSQL database
  • don’t carry data certified as Secret or above
  • write applications in a language supported by the default Cloud Foundry buildpacks
